# 飼い主向け受診サポートツール MVP 定義書

## プロジェクト目的
飼い主の曖昧な違和感を言語化し、獣医師に正確に伝えるためのレポート作成を支援する。
※注意：診断は行わず、情報の整理のみに特化する。

## PO判定による機能選定

### 1. 【MVPに必須】（Must）
最小の機能で動作するプロトタイプとして、まずは以下の機能を実装します。

* **ペットの基本情報入力**: 種別（犬/猫等）、年齢、性別、体重、既往歴などの簡易フォーム。
* **構造化された症状ヒアリング**: 
  * いつから（発症時期）
  * 食欲・元気・排泄状況（選択式）
  * 主訴となる症状（選択肢）
* **曖昧な違和感の自由記述欄**: 飼い主の言葉でそのまま記載できるテキストエリア。
* **受診用レポート生成（閲覧・コピー）**: 入力された情報を、獣医師が短時間で把握しやすい構造化テキスト（箇条書き）として画面に表示し、ワンタップでクリップボードにコピーできる機能。

### 2. 【後回し】（Postpone）
最短リリースと最小コストを優先し、以下の機能は初期MVPスコープから除外します。

* **写真・動画のアップロード機能**: クラウドストレージ構築のコストと時間がかかるため。まずはテキストレポートの有用性を検証します。
* **アカウント登録・過去履歴保存機能**: 認証基盤やDB構築による複雑化を避けるため。MVPは「その場完結のツール」とします。
* **AIによる対話型の深掘りチャット**: 意図せず診断に近い発言をするリスクを排除するため。まずは静的かつ安全な構造化フォームで要件を満たします。
* **PDFエクスポート / かかりつけ医のシステムへの直接送信**: 実装コストが高いため。クリップボードコピーを活用して飼い主自身が利用・共有（LINE等）する想定にします。

### 3. 【不要】（Unnecessary）
コンセプト（情報の整理に特化）から逸脱するため、以下の機能は永久に実装しません。

* **病名の予測・診断アルゴリズム**: 獣医師法に抵触するリスクがあるため。
* **特定の市販薬・対処法のレコメンド機能**: 同上。
* **オンラインの獣医師相談・マッチング**: 本質は「実際の動物病院を受診する際のサポートツール」であるため。
